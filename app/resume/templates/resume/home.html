What I Like About Software Development


Efficiency

I find it interesting when people are able to create incedibly efficient code. For example, Doom 1993 was revolutionary at the time it was released because the programmers, primarily John Carmack, utilized binary space partitioning which had only been theorized and never actually applied to a real program. Even something as simple as using checkerboard rendering to increase video game resolution while utilizing the same processing power is amazing.

Bug Fixing

Despite being extremely frustrating at times, bug fixing can be quite enjoyable for me. Something about hyper-focusing on one issue can be relaxing. The feeling of relief when you finally figure it out is pretty great as well.

Always Learning

Software development is an ever evolving industry. Languages, frameworks, and tools constantly get updated. New tech is always being developed. With a career in software there's always something new to learn and adapt to. It's nice to know that even if I'm working the exact same job for 20 years the day-to-day work will change.
